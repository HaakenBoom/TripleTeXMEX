# Tripletex API Reference (from OpenAPI spec)

## Response formats
- Single entity: {"value": {...}}
- List: {"fullResultSize": N, "values": [...]}
- Error: {"status": 4xx, "message": "...", "validationMessages": [...]}
- Reference fields: {"id": <int>} — NEVER raw strings
- Dates: YYYY-MM-DD format

## Ledger Accounting Dimension
  GET /ledger/accountingDimensionName — Get all dimension names
  POST /ledger/accountingDimensionName — Create a new custom dimension
  GET /ledger/accountingDimensionName/search — Search dimensions
  GET /ledger/accountingDimensionName/{id} — Get by ID
  PUT /ledger/accountingDimensionName/{id} — Update dimension
  DELETE /ledger/accountingDimensionName/{id} — Delete dimension
  Query: dimensionIndex, activeOnly, onlyDimensionsWithActiveValues, fields, count, from
  Body fields (AccountingDimensionName):
    dimensionName: string — The name of the dimension
    description: string
    active: boolean

  POST /ledger/accountingDimensionValue — Create a value for a custom dimension
  GET /ledger/accountingDimensionValue/search — Search dimension values
  GET /ledger/accountingDimensionValue/{id} — Get value by ID
  DELETE /ledger/accountingDimensionValue/{id} — Delete value
  Query: dimensionIndex, activeOnly, showInVoucherRegistration, fields, count, from
  Body fields (AccountingDimensionValue):
    displayName: string — The name of the value
    dimensionIndex: int32 — Which dimension this value belongs to (1, 2, or 3)
    active: boolean
    number: string — Code/number for the value
    showInVoucherRegistration: boolean

  NOTE: Voucher postings can link to dimensions via freeAccountingDimension1/2/3 fields
  on Posting objects, using {"id": <dimensionValueId>}
